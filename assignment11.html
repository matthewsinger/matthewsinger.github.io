Assignment 11: Recipes

Goals: Practice the visitor pattern.

You should submit one .java file containing the solution to this problem.

Be sure to properly test your code and write purpose statements for your methods. A lack of tests and documentation will result in a lower grade! Remember that testing requires you to make some examples of data in an examples class.

Recipes

Below is a tree-shaped way of modeling a recipe:
// a recipe interface IRecipe {}
 
// a base ingredient class Ingredient implements IRecipe {
String name;
String unitOfMeasure;
double amount;
 
Ingredient(String name, String unitOfMeasure, double amount) {
this.name = name;
this.unitOfMeasure = unitOfMeasure;
this.amount = amount;
}
}
 
// transform a recipe (such as heating, chopping, etc.) class Transform implements IRecipe {
String instruction;
IRecipe base;
 
Transform(String instruction, IRecipe base) {
this.instruction = instruction;
this.base = base;
}
}
 
// mixing a recipe (such as blending, pouring one over the other, etc.) class Mix implements IRecipe {
String instruction;
IRecipe r1;
IRecipe r2;
 
Mix(IRecipe r1, IRecipe r2, String instruction) {
this.instruction = instruction;
this.r1 = r1;
this.r2 = r2;
}
}

  1. Enrich the IRecipe interface with the visitor pattern. The accept method is the only method that should be added to the interface in this assignment.

  2. Determine whether or not a given ingredient (given by just the name) appears in a recipe.

  3. Double the amount of every ingredient in a recipe.

  4. Produce a list of every ingredient in the recipe. Optional: If the same ingredient is measured by the same units in more than one place in the tree, combine them to just be one output in the list (with their amounts totalled).
