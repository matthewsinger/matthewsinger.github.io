Assignment 20: Flood It

Goals: Design a game with mutable world state.

You should submit one .java file containing the solution to this problem.

Be sure to properly test your code and write purpose statements for your methods. A lack of tests and documentation will result in a lower grade! Remember that testing requires you to make some examples of data in an examples class.

You will be using the Impworld library. To import it, include the following at the top of your file:
import tester.*;
import javalib.impworld.*;
import java.awt.Color;
import javalib.worldimages.*;

Setting the scene

In this game players are trying to manipulate a grid of colors so that all of the cells are the same color. The player starts in the upper left corner and by clicking various colors they can change the color of that corner which can increase the size of the area they control.

An example of this game is available for you to play here. The game essentially has two states, one which allows you to click on a tile to begin flooding, and another which is in the process of flooding.

Requirements

  • Your game should take in a grid size and the number of colors that will be used. Use these numbers to produce your initial world.

  • All cells should be initialized with a random color.

  • If a player clicks on a square that is the same color as the area they control, no change should occur.

  • If a player clicks on a square that is distinct from the color of the area they control, this produces the "waterfall" effect.

Hints

  • You may assume the number of colors will not exceed 8.

  • When you are in the process of flooding, your on-tick method should flood only the squares adjacent to those you have already flooded. This will produce the "waterfall" effect shown in the game.

  • Once one round of flooding is completed, consider all cells unflooded.

  • Flooding can happen in any of the cardinal directions, not just right and down.

  • First, design your game’s constructor. Second, design and verify the accuracy of your graphics. Save the actual gameplay for last.

  • You are welcome to enhance the game outside of the requirements in this assignment, but only do so after you have completed what is required.
