Assignment 21: Final Project

Goals: Design a complex program using all we have learned.

Be sure to properly test your code and write purpose statements for your methods. A lack of tests and documentation will result in a lower grade! Remember that testing requires you to make some examples of data in an examples class.

Project requirements

For your final project, you need to produce a sufficiently complex, complete application. This application should serve to either entertain, solve a problem, answer a question, or some combination of the above.

For the entertainment route, games on the complexity order of centipede, tower defense games, text-based adventure games, and 2D-platformers are appropriate. Note that if your gameplay is simple, ways to make your game more interesting include menus, graphics, lives, settings, etc.

As for solving a problem, creating anything similar to the products offered by Google (Drive, Docs, Presentations, Sheets, Calendar, etc.) would be more than sufficiently complex.

For answering a question, you can use the algorithms we are learning about to answer an actual question about an actual data set.

For all of the above options, use of I/O to load an initial state, read in data, save an intermediate state, or write out an answer, are all ways to make your program more interesting and useful.

The general guideline for "sufficiently complex" means that implementing your program should be non-obvious and take a serious amount of planning as well as programming time. Note that a relatively simple application that is well-designed to allow for fast and easy extension (such as an additional level) is also an admirable goal.

You do not have to mimic an existing game, product, or algorithm, but can invent your own, so long as it meets the above description.

Technical requirements

The only technical requirements for the project is you must implement it in Java, and you must use the tester library to test it. You are welcome to use the big bang library or not. Note that if you are using another library, I cannot guarantee assistance, and you must provide instructions for how to use and a way to access the library. You are welcome to use any library that does not trivialize the problem, such as importing a physics engine library and then writing a game where one just bounces a ball.

A word of caution

This is the first assignment all year where I do not already have a solution in mind to guide you to. Documenting your code, testing it as you go, and breaking it up into helpers is essential for success.

Project outline

The project consists of the following major miletones:

  • Project proposal - Monday, March 30th

  • Class heiarchy - Monday, April 6th

  • Check-in - Wednesday, April 22nd

  • Submission + Presentation - Wednesday, March 20th or Wednesday, March 27th

Project proposal

Your proposal should be a brief (1-3 paragraphs) overview of what you plan on making. Be sure to convey its main purpose, how one will interact with it, and what you anticipate will be difficult. If helpful and possible, provide links to inspiration, or include sketches of what your product might look like. If your proposal is sufficiently complex, then you will be given an immediate thumbsup to proceed. If not, we will meet to discuss how to enhance it. In either case, I will provide some initial, high-level suggestions that come to mind as to how to structure your code.

Class heiarchy

Submit a Java file (or archive) of the various classes and interfaces that are core to your project. Classes should include their fields, the classes they are extending, and interfaces they are implementing (if any). Interfaces should include method signatures. All classes and methods should include at least some documentation, though actual code and testing are not necessary at this point. Feedback will be given after this point.

Check-in

Submit all of the code (with aforementioned references to external libraries if applicable) you have completed thusfar, even if it is in a currently buggy or incomplete state. The purpose of this is to get feedback on your work so far.

Submission + Presentation

Submit all of your code and give a ten minute presentation to the class. Your presenetation should give an overview of your product, a demo, a discussion of the programming strategies you used, and brief snippets of particularly interesting or complex code. Please create your presentation on Google Drive, and discuss how the demo will be best coordinated with me ahead of time. Be sure to put some effort into your presentation, as they are both a reflection of the hard work you’ve done and an opportunity to inspire and teach your peers (and teacher!).

Grading

Your grade will largely be based on the quality of your code, as well as how functional your final product is. Note that code quality includes what it always has: readability, documentation, and testing. Additionally, the content of your presentation will play some role.
