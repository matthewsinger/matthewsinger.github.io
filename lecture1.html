Lecture 1: Welcome

Motivation: We want to program

Topics to cover: Welcome, DrRacket, definitions window, repl, stepper, numbers, strings, expressions, comments

Welcome to DrRacket!

DrRacket is an IDE, or integrated development environment. This is basically a fancy expression which means DrRacket is a tool that will make programming easy for us, since we are now all developers.

The programming language we will use, appropriately, is Beginning Student Language. To set this as your language, click Language > Choose Language > Beginning Student > OK.

The three tools

DrRacket comes with three main tools as far as we are concerned.

One is the definitions window. That’s the big, large, empty white rectangle.

The second is the interactions window. That’s the window below the definitions window, with a carrot.

The third is the stepper, which is launched by pressing the "step" button above.

Today, we’re going to learn about these three tools, comments, numbers, strings, and expressions.

Let’s start with comments. Comments are invaluable, and we will talk much more about them as the year progresses.

This is a block comment, or a comment that spans multiple lines. It is started with a hashpipe and ends with a pipehash, e.g. #| and |#.

A single line comment starts with a semicolon:

; like this

Numbers, the interactions window, expressions

As you might expect, BSL is able to compute numbers. Let’s try my favorite number, 42.

When we type 42 next to the carrot in the interactions window below and hit enter: what happens?

It spits back out the number 42. How about 5.3? Or -2? Or 6/7?

We’ve now hopefully learned two things: we can type all sorts of numbers, and when we enter one in the interactions window, it spits it back out.

But why is this? As it turns out, the interactions window evaluates expressions. Our expressions will get bigger and more complicated over the year, but they can be as simple as just a number. An expression is any valid BSL statement.

Let’s try a slightly more interesting expression, using our friend, arithmetic.

As you probably know, 5 + 3 is equal to 8. It seems like computers should know how to compute this, and BSL (or beginning student language) does in fact know how to. Lets see how to program this in BSL, by entering the following into the interactions window: (+ 5 3).

Yay, we get 8, as expected. We now have another crucial insight: in BSL, operations, which we will call functions, go to the left of an open parenthesis, followed by arguments to the function, followed by a closing parenthesis.

Note that there are two common ways to express this in english: we can say we are calling + on 5 and 3, or we can say we are passing 5 and 3 to +.

Let’s try a more complex expression. How would we convert (5 - 3 + 4) * 8 into a BSL expression? Evaluating by hand, first, we see that the result would be 48, so whatever we write should get us 48.

Well, first we have to identify the outermost operation. That’s going to go into the first set of open parenthesis, as it will occur last in the evaluation.

The outermost operation, as we know from PEMDAS, is *. So we know our answer has to be of the form (* _ _). The right hand side is pretty easy to fill in, it’s just 8. So now we have (* _ 8).

But how do we convert 5 - 3 + 4 into a BSL expression? Well, we need to identify the outermost operation: what happens last? The +. So now, we know our answer is of the form (* (+ _ _) 8). The last blank is easy to fill in, it’s just 4.

So now we have (* (+ _ 4) 8). And what goes in the first blank? Whatever the BSL version of 5 - 3 is, which is (- 5 3). So now, we have (* (+ (- 5 3) 4) 8). When we evaluate this in the interactions window, we get 48, which is expected.

As we have seen, the interactions window is a way to test expressions. This will be very helpful throughout the year.

The stepper and definitions window

But how does this expression get evaluated? For that, we’ll use the stepper. Let’s type (* (+ (- 5 3) 4) 8) in the definitions window, which is the big rectangle above the interactions window.

Now, hit "step" in the upper right hand corner.

The stepper shows you how an expression is evaluated, step by step. Expressions are evaluated left-to-right and innermost first.

So (* (+ (- 5 3) 4) 8) is evaluated as follows:

(* (+ (- 5 3) 4) 8)

(* (+ 2 4) 8)

(* 6 8)

48

Notice that the computation stops once we reach 48. That is because the program has reached a value, and returns it. For now, the only kind of value we know about is numbers, but that will soon change.

Let’s click "run" now. As we can see, the 48 is output in the interactions window.

When we run the definitions window, all expressions are evaluated and printed out in the interactions window.

When we add another expression to the interactions window and hit run, we them all both below.

Let’s try it out on these expressions.

(sqr 3)

(sqrt 9)

(expt 2 3)

(- 3)

(/ 1 2)

Also, some functions, like + and *, can take an arbitrary number of elements. For example, (* 1 2 3 4).

Strings

While numbers are great, we also love words. Words are represented as Strings, which are contained without double quotes. Let’s type "Hello World" and hit enter in the interactions window. We get "Hello World" back. Strings, therefore, are another kind of value.

We also have functions that operate on strings. For now, three will be of interest:

string-length, which takes one string and outputs its length (e.g. (string-length "cat"))

string-append, which takes an arbitrary number of strings and appends them together. (e.g. (string-append "cat" "dog"))

substring, which takes a string and two indices and outputs that part of the string. For example, let’s run (substring "apple" 2 4) in the interactions window. Why do you think the output is "pl"?
