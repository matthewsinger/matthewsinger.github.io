posn

So let’s say we wanted to make an animation wherea circle moved both horizontally and vertically. What kind of data would we need to keep track of? The x and y coordinate.

This is such a common need for big-bang programs that this structure comes pre-installed, and it’s called posn. It has an x and y field:

; (define-struct posn [x y])

We leave it commented out because it’s already pre-installed, but it still warrants a data definition because it’s a complex data type.

; A Posn is a (make-posn Number Number)
; (define-struct posn [x y])
; and represents a coordinate on the cartesian plane
 
(define posn-origin (make-posn 0 0))
(define posn-1/2 (make-posn 1 2))
 
; posn-temp : Posn ->
(define (posn-temp p)
  (... (posn-x p) (posn-y p)))

Let’s write a simple function that adds two positions to each other.

; add-posns : Posn Posn -> Posn
; Add the x and y coords of each posn
(define (add-posns p1 p2)
  (make-posn (+ (posn-x p1) (posn-x p2)) (+ (posn-y p1) (posn-y p2))))
(check-expect (add-posns (make-posn 1 2) (make-posn 3 4)) (make-posn 4 6))

So let’s begin our program with a basic setup.

(define DOT (circle 20 "solid" "blue"))
(define BG (empty-scene 200 200))
 
; move-dot : Posn -> Posn
; Move a dot around the screen
(define (move-dot p0)
  (big-bang p0
    [to-draw place-at]))
 
; place-at : Posn -> Image
; place DOT on BG at p
(define (place-at p)
  (place-image DOT (posn-x p) (posn-y p) BG))
(check-expect (place-at posn-1/2) (place-image DOT 1 2 BG))

How about, instead of moving it over time, we put it where we click.

on-mouse

Our next big bang handler will be

on-mouse

. The on moust function has the following signature

; on-mouse-function : WorldState Nat Nat MouseEvent -> WorldState
(define (on-mouse-function past-state x y me) ...)

The two numbers represent the x and y positions of the mouse event. Let’s define a MouseEvent.

; A MouseEvent is one of:
; - "leave" (mouse leaves the window)
; - "enter" (mouse enters the window)
; - "move" (mouse is moved in the window)
; - "drag" (mouse is dragged in the window)
; - "button-down" (mouse has been clicked)
; - "button-up" (mosue click has been released)

Almost all of the time, you’ll care about "button-down". Let’s move our dot when we click to where we clicked.

(define DOT (circle 20 "solid" "blue"))
(define BG (empty-scene 200 200))
 
; move-dot : Posn -> Posn
; Move a dot around the screen
(define (move-dot p0)
  (big-bang p0
    [to-draw place-at]
    [on-mouse click]))
 
; place-at : Posn -> Image
; place DOT on BG at p
(define (place-at p)
  (place-image DOT (posn-x p) (posn-y p) BG))
(check-expect (place-at posn-1/2) (place-image DOT 1 2 BG))
 
; click : Posn Nat Nat MouseEvent -> Posn
; Update the posn to the place of a click
(define (click p x y me)
  (cond [(string=? me "button-down") (make-posn x y)]
        [else p]))
(check-expect (click posn-1/2 3 4 "button-down") (make-posn 3 4))
(check-expect (click posn-1/2 3 4 "move") posn-1/2)

While the program is very simple here, you want your on-mouse function to follow the MouseEvent template (i.e., a cond where you check which mouse event it is), and then call a helper if it does anything complex.

What’s the other way, besides the mouse, you usually interact with the computer?

on-key

The handler for key events is called on-key, and the signature is:

; on-key-function : WorldState KeyEvent -> WorldState

Most key events are just the literal key, like "a" or "1". Some are special though, like "left" for the left key. (show docs where it describes all of them)

Let’s move our dot based on key inputs.

(define DOT (circle 20 "solid" "blue"))
(define BG (empty-scene 200 200))
 
; move-dot : Posn -> Posn
; Move a dot around the screen
(define (move-dot p0)
  (big-bang p0
    [to-draw place-at]
    [on-mouse click]
    [on-key shift]))
 
; place-at : Posn -> Image
; place DOT on BG at p
(define (place-at p)
  (place-image DOT (posn-x p) (posn-y p) BG))
(check-expect (place-at posn-1/2) (place-image DOT 1 2 BG))
 
; click : Posn Nat Nat MouseEvent -> Posn
; Update the posn to the place of a click
(define (click p x y me)
  (cond [(string=? me "button-down") (make-posn x y)]
        [else p]))
(check-expect (click posn-1/2 3 4 "button-down") (make-posn 3 4))
(check-expect (click posn-1/2 3 4 "move") posn-1/2)
 
; shift : Posn KeyEvent -> Posn
; Shift the posn based on input
(define (shift p ke)
  (cond [(string=? ke "left") (add-posns p (make-posn -5 0))]
        [(string=? ke "right") (add-posns p (make-posn 5 0))]
        [(string=? ke "up") (add-posns p (make-posn 0 -5))]
        [(string=? ke "down") (add-posns p (make-posn 0 5))]
        [else p]))
(check-expect (shift posn-1/2 "left") (make-posn -4 2))
(check-expect (shift posn-1/2 "right") (make-posn 6 2))
(check-expect (shift posn-1/2 "up") (make-posn 1 -3))
(check-expect (shift posn-1/2 "down") (make-posn 1 7))
(check-expect (shift posn-1/2 "a") posn-1/2)
