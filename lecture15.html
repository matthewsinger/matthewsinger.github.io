Lecture 15: Self-referential Data

Motivation: We want to model recursive data

Topics to cover: self referential data, templates, functions, empty-image

Recap

Any insights from the homework? What didn’t people understand?

Matroyshka Dogs

The lecture notes refer to these dolls.

Show the dog and how it’s nested, down to the bone at the core. Say all dogs have a name. On white board, ask students how they’d design a data definition for this. Develop something akin to:

; A ND (NestedDog) is one of:
; - 'bone
; - (make-dog String ND)
(define-struct dog [name inside])
; and represents either the bone at the center or a dog with
; a name and something inside it

Draw the arrow between ND and ND.

So, let’s put this into BSL and develop examples and a template.

;           
; A ND (NestedDog) is one of:
; - 'bone             
; - (make-dog String ND)
(define-struct dog [name inside])
; and represents either the bone at the center or a dog with
; a name and something inside it
 
(define nd-bone 'bone)
(define dog1 (make-dog "abe" nd-bone))
(define dog2 (make-dog "bob" nd-bone))
(define dog3 (make-dog "carl" nd-bone))
(define dog4 (make-dog "dave" nd-bone))
 
; nd-temp : ND -> ?
(define (nd-temp nd)
  (cond [(symbol? nd) ...]
        [(dog? nd) (... (dog-name nd) (nd-temp (dog-inside nd)))]))

Inside of a dog, there’s a string and an ND. An ND is a complex data type, so we need to call its template: which is this template! Notice the reptition matches EXACTLY where it’s repeated in the data definition!

Notice that this isn’t introducing any new rules for a template: we’ve been calling templates of complex data types inside templates. It’s just that, for the first time, that template is already the one we’re defining.

Let’s design a function which determines how many dogs there are given an ND.

; number-of-dogs : ND -> Nat
; How many dogs are in this ND
(define (number-of-dogs nd)
  (cond [(symbol? nd) 0]
        [(dog? nd) (add1 (number-of-dogs (dog-inside nd)))]))
 
(check-expect (number-of-dogs nd-bone) 0)
(check-expect (number-of-dogs dog1) 1)
(check-expect (number-of-dogs dog2) 2)
(check-expect (number-of-dogs dog3) 3)
(check-expect (number-of-dogs dog4) 4)

Let’s design a function which determines if any dog is named a given string.

; any-dog-named? : ND String -> Boolean
; Is any dog named name?
(define (any-dog-named? nd name)
  (cond [(symbol? nd) #f]
        [(dog? nd) (or (string=? (dog-name nd) name) (any-dog-named? (dog-inside nd) name))]))
 
(check-expect (any-dog-named? nd-bone "carl") #f)
(check-expect (any-dog-named? dog1 "carl") #f)
(check-expect (any-dog-named? dog2 "carl") #f)
(check-expect (any-dog-named? dog3 "carl") #t)
(check-expect (any-dog-named? dog4 "carl") #t)

Let’s design a function which replaces each instance of a given name with another given name.

; replace-names : ND String String -> ND
; Replace all instances of name1 with name2
(define (replace-names nd name1 name2)
  (cond [(symbol? nd) nd]
        [(dog? nd) (make-dog
                    (if (string=? (dog-name nd) name1) name2 (dog-name nd))
                    (replace-names (dog-inside nd) name1 name2))]))
 
(check-expect (replace-names nd-bone "carl" "clause") nd-bone)
(check-expect (replace-names dog1  "carl" "clause") dog1)
(check-expect (replace-names dog2  "carl" "clause") dog2)
(check-expect (replace-names dog3  "carl" "clause") (make-dog "clause" dog2))
(check-expect (replace-names dog4  "carl" "clause") (make-dog "dave" (make-dog "clause" dog2)))

Let’s design a function which removes all instances of a name.

; remove-all-names : ND String -> ND
; Remove all dogs with a certain name
(define (remove-all-names nd name)
  (cond [(symbol? nd) nd]
        [(dog? nd) (if (string=? (dog-name nd) name)
                       (remove-all-names (dog-inside nd))
                       (make-dog (dog-name nd) (remove-all-names (dog-inside nd))))]))
 
(check-expect (remove-all-names nd-bone "carl") nd-bone)
(check-expect (remove-all-names dog1  "carl") dog1)
(check-expect (remove-all-names dog2  "carl") dog2)
(check-expect (remove-all-names dog3  "carl") dog2)
(check-expect (remove-all-names dog4  "carl") (make-dog "dave" dog2))

Let’s design a function which produces an image of the dog names above each other. What makes sense for a base case? Nothing! There exists a constant empty-image, which is a 0x0 image.

; names-picture : ND -> Image
; Produce a picture of all the names
(define (names-picture nd)
  (cond [(symbol? nd) empty-image]
        [(dog? nd) (above (text (dog-name nd) 20 "green")
                          (names-picture (dog-inside nd)))]))
 
(check-expect (names-picture nd-bone) empty-image)
(check-expect (names-picture dog1) (text "abe" 20 "green"))
(check-expect (names-picture dog2) (above
                                    (text "bob" 20 "green")
                                    (text "abe" 20 "green")))
(check-expect (names-picture dog3) (above
                                    (text "carl" 20 "green")
                                    (text "bob" 20 "green")
                                    (text "abe" 20 "green")))
(check-expect (names-picture dog4) (above
                                    (text "dave" 20 "green")
                                    (text "carl" 20 "green")
                                    (text "bob" 20 "green")
                                    (text "abe" 20 "green")))
