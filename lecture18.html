A standard way to encode lists

So we’ve written many kinds of collections of data at this point: to represent buildings, skylines, positions, and more. But everytime we need to develop a new structure. Let’s standardize this.

First, there’s the base case. This we will call empty. Let’s check it out in the interactions window. Note that it prints '(); this is another way to write empty.

We can also ask if any value is empty?, and this will only return #t for empty. Let’s check it out in the interactions window.

Next is the actual data. It has to have what? Data, and the recursive bit. The thing that will hold these two values is called a cons cell, short for constructs, and cons is the name of the function that takes the two things. Let’s try it out in the interactions window. We can create lists of arbitrary size.

But what happens if we try to construct a list without cons or empty as the second argument? It blows up, which is nice, so we can only ever construct well-formed lists.

How do we get the data out of a cons cell? With first and rest. Let’s try it out in the interactions window.

You can also identify a cons cell with cons?.

You can think of empty and cons as if the following structures were made, but with some replacements:

; (define-struct empty [])
; (make-empty) replaced with empty
; empty? as is
; no selectors
 
; (define-struct cons [first rest])
; (make-cons a b) replaced with (cons a b)
; cons? as is
; (cons-first (make-cons a b)) -> a replaced with (first (cons a b)) -> a
; (cons-rest (make-cons a b)) -> b replaced with (rest (cons a b)) -> b

These constants and functions are so common that writing cons-first and cons-rest and make-cons and make-empty every time would be so tedious that it’s nice we don’t have to.

So, how do we construct a data definition with this? Let’s say we’re defining a list of numbers.

; A LoN (List of Numbers) is one of:
; - empty
; - (cons Number LoN)
 
(define lon0 empty)
(define lon1 (cons 1 lon0))
(define lon2 (cons 3 lon0))
(define lon3 (cons 4 lon0))
 
; lon-temp : LoN -> ?
(define (lon-temp lon)
  (cond [(empty? lon) ...]
        [(cons? lon) (... (first lon) (lon-temp (rest lon)))]))
