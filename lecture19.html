An interesting list

Let’s think of what happens at a grocery store. When someone scans a piece of food, a price per unit comes up. Otherwise, it’s measured by weight. As you can imagine, grocery stores need to keep a lot of these pieces of information around. Let’s come up with a data definition to represent this.

; A GC (GroceryCatalogue) is one of:
; - empty
; - (cons CE GC)
 
; A CE (CatalogueEntry) is a (make-ce Symbol Cost)
(define-struct ce [food cost])
; and represents the name of a food and its associated cost
 
; A Cost is one of:
; - (make-unit NNN)
(define-stuct unit [value])
; - (make-lb NNN)
(define-struct lb [value])
; and represents the cost of food, whether by unit or by value
 
(define cost10u (make-unit 10))
(define cost5lb (make-lb 5))
 
(define banana (make-ce 'banana cost10u))
(define apple (make-ce 'apple cost5lb))
 
(define gc0 empty)
(define gc1 (cons banana gc0))
(define gc2 (cons apple gc1))
 
; gc-temp : GC -> ?
(define (gc-temp gc)
  (cond [(empty? gc) ...]
        [(cons? gc) (... (ce-temp (first gc)) (gc-temp (rest gc)))]))
 
; ce-temp : CE -> ?
(define (ce-temp ce)
  (... (ce-food ce) (cost-temp (ce-cost ce))))
 
; cost-temp : Cost -> ?
(define (cost-temp c)
  (cond [(unit? c) (... (unit-value c))]
        [(lb? c) (... (lb-value c))]))

Now, what’s something a cashier would obviously need to do? Look up a cost by food. Let’s design that function. What’s an obvious edge case, and how should we handle it? (missing, error)

; lookup : GC Symbol -> Cost
; Lookup the cost of an item by name
(define (lookup gc food)
  (cond [(empty? gc) (error (string-append (symbol->string food)) " was not found")]
        [(cons? gc) (if (matches? (first gc) food) (first gc) (lookup (rest gc) food))]))
(check-error (lookup gc2 'grape) "grape was not found")
(check-expect (lookup gc2 'banana) cost10u)
(check-expect (lookup gc2 'apple) cost5lb)
 
; matches? : CE Symbol -> Boolean
; Does the entry match food?
(define (matches? ce food)
  (symbol=? food (ce-food ce)))
(check-expect (matches? banana 'banana) #t)
(check-expect (matches? apple 'banana) #f)

Another thing that might need to happen is we may need to update or add a new food to the catalogue.

; update : GC Symbol Cost -> GC
; Update the catalogue, associating food with cost
(define (update gc food cost)
  (cond [(empty? gc) (cons (make-ce food cost) empty)]
        [(cons? gc) (if (matches? (first gc) food)
                        (cons (make-ce food cost) (rest gc))
                        (cons (first gc) (update (rest gc) food cost)))]))
(check-expect (update gc1 'apple cost5lb) (cons banana (cons apple empty)))
(check-expect (update gc2 'banana cost5lb) (cons apple (cons (make-ce banana cost5lb) empty)))

Groceries

The other thing that a cash register needs to work is a list of groceries. Some groceries are just a unit, but others are weighed. How might we represent that?

; A Checkout is one of:
; - empty
; - (cons Grocery Checkout)
 
; A Grocery is one of:
; - Symbol
; - (make-weight Symbol Number)
(define-struct weight [name lb])
; and represents either one unit of food or its name and weight in lbs
 
(define grocery/b 'banana)
(define grocery/a (make-weight 'apple 2))
 
; checkout-temp : Checkout -> ?
(define (checkout-temp c)
  (cond [(empty? c) ...]
        [(cons? c) (... (grocery-temp (first c)) (checkout-temp (rest c)))]))
 
; grocery-temp : Grocery -> ?
(define (grocery-temp g)
  (cond [(symbol? g) ...]
        [(weight? g) (... (weight-name g) (weight-lb g))]))
