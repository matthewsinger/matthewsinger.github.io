Let’s design a function which given a list of posns draws dots at all of those locations on an empty scene. Why should we use?

(define DOT (circle 5 'solid 'red))
(define BG (empty-scene 200 200))
 
; draw-dots-onto : [List-of Posn] -> Image
; Draw DOT at ps onto BG
(define (draw-dots-onto ps)
  ; How to decide what goes here?
  ; foldr : (X Y) [X Y -> Y] Y [List-of X] -> Y
  ;           ^
  ; What is X, what is Y?
  ; foldr : [Posn Image -> Image] Y [List-of Posn] -> Image
  ;          ^ need to make      ^bg ^ps
  (foldr draw-dot-onto-i-at-p BG ps))
(check-expect (draw-dots-onto empty) BG)
(check-expect (draw-dots-onto (list (make-posn 5 6) (make-posn 9 10)))
              (place-image DOT 5 6 (place-image DOT 9 10 BG)))
 
; draw-dot-onto-i-at-p : Posn Image -> Image
; Draw DOT onto i at p
(define (draw-dot-onto-i-at-p p i)
  (place-image DOT (posn-x p) (posn-y p) i))
(check-expect (draw-dot-onto-i-at-p (make-posn 3 4) BG) (place-image DOT 3 4 BG))

The general process for using these and any other abstractions is to follow the signature all the way through.
