Lecture 27: Local

Motivation: We want to use pre-defined abstractions based on input

Topics to cover: local, scope, local definitions

Recap

Any insights from the homework? What didn’t people understand?

Getting stuck

Let’s say we want to define a function that removes everything less than a certain number from a list of numbers. Which abstraction should we use?

; remove-less-than : [List-of Number] Number -> [List-of Number]
; Remove all numbers less than n
(define (remove-less-than lon n)
  (filter ?!?! lon))
(check-expect (remove-less-than (list 1 2 3) 2) (list 2 3))

We’re stuck, because we need a function that says whether or not something is bigger than or equal to n.

But that function doesn’t already exist. How do we create functions? With define. But define only works at the top level!

Well, it did only work at the top level...

Enter local.

Local

local is a new piece of syntax that lets us define values locally.

For example, running (local [(define x 3)] x) will get us 3. But if we try and run just x after that, we get an error that x is not defined. This will also happen in the definitions window.

(local [(define x 3)] x)
x
Let’s make this a bit more complex... what about:

(local [(define x 3)
        (define y (+ x 4))]
  (local [(define x y)]
    (+ y 10 x)))
(local [(define x 3)
        (define y (+ x 4))]
  (local [(define x
            (local [(define y x)]
              y))]
    (+ y 10 x)))

Why doesn’t this work? Because it’s effectively like going (define x x).

So, back to our original problem: how can local help us?

; remove-less-than : [List-of Number] Number -> [List-of Number]
; Remove all numbers less than n
(define (remove-less-than lon n)
  (local [; <=n : Number -> Boolean
          ; Is n less than or equal x?
          (define (<=n x)
            (<= n x))]
    (filter <n lon)))
(check-expect (remove-less-than (list 1 2 3) 2) (list 2 3))

Note that when we define local functions we add a signature and purpose statement, but what’s missing? Tests. We cannot check-expect a local function, so all of our local functions should be obvious to the point they don’t need tests. They’ll get implicitly tested by testing the body of the whole function.

What kind of definition haven’t we tried in a local yet?

(local [(define foo [a b])] (make-foo 1 2))

What can I do with this? Nothing. I can’t even call foo?.

Note that
(equal? (local [(define foo [a b])] (make-foo 1 2))
        (local [(define foo [a b])] (make-foo 1 2)))
doesn’t even return true. Why do you think this is?

What about

(local [(define pair (local [(define-struct foo [a b])]
                       (list (make-foo 1 2)  (make-foo 1 2))))]
  (equal? (first pair) (second pair)))

Do you think it will?

So that’a fun thought experiment. Let’s learn one more new thing: build-list.

build-list

build-list takes in a natural number and a function, and then applies it to all of those numbers up through one less than the number. Let’s experiment in the interactions window.

(build-list 0 identity)
(build-list 10 identity)
(build-list 10 add1)
(build-list 10 sqr)

Now let’s try using build-list and local to define a function between that produces a list of numbers between two integers.

; between : Integer Integer -> [List-of Integer]
; all numbers between a (inclusive) and b (exclusive)
(define (between a b)
  (local [; add-a : Nat -> Integer
          ; Add a to i
          (define (add-a i)
            (+ a i))]
  (build-list (max (- b a) 0) add-a)))
(check-expect (between 1 1) (list))
(check-expect (between 1 4) (list 1 2 3))
(check-expect (between 1 -4) (list))
(check-expect (between -4 4) (list -4 -3 -2 -1 0 1 2 3))
