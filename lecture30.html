Lecture 30: Functions that make functions

Motivation: We want to make functions on the fly easily

Topics to cover: Functions that produce functions

Recap

Any insights from the homework? What didn’t people understand?

Producing a function

Let’s say we want a function that adds 3 to a number.

; add3 : Number -> Number
; Add 3 to n
(define (add3 n) (+ 3 n))
(check-expect (add3 4) 7)

Now let’s say we want to add 5.

; add5 : Number -> Number
; Add 5 to n
(define (add5 n) (+ 5 n))
(check-expect (add3 4) 9)

If we just abstracted this in the normal way, what would we get? A binary version of +. Boring, and unhelpful!

What do we really want here? We want something that takes a number and produces a function that will add it to a number.

; adder : Number -> ?
(define (adder n)
  (local [; add-n : Number -> Number
          ; Add n to x
          (define (add-n x)
            (+ x n))]
    add-n))

So what is the type, and how do we test this?

; adder : Number -> [Number -> Number]
(define (adder n)
  (local [; add-n : Number -> Number
          ; Add n to x
          (define (add-n x)
            (+ x n))]
    add-n))
(check-expect ((adder 3) 4) 7)

So what is the type, and how do we test this? It blows up! We could name it like this:

; adder : Number -> [Number -> Number]
(define (adder n)
  (local [; add-n : Number -> Number
          ; Add n to x
          (define (add-n x)
            (+ x n))]
    add-n))
(define add3 (adder 3))
(check-expect (add3 4) 7)

If we look at the ISL docs, this makes sense, because it requires a name after a function call, not an expression. But if we have expresisons that can produce functions, that’s very restricting.

Everyone stand up, because we are graduating yet again!

ISL+

We are changing out language to intermediate student with lambda, where our original check-expect works.

But this begs the question: what is lambda?

lambda is a way of making anonymous function. Anonymous function effectively means nameless function.

Before, we had functions look like (define (name args...) body), but now we can write functions that look like (lambda (args...) body)

Let’s look at an example.

(λ (x) (+ x 2)). This is a function that takes one thing and adds 2 to it. λ is equivalent to lambda and can be entered with command + backslash (or Racket > Insert Lambda). Either is fine, but I use λ to save space.

(λ (x y z) (= x (string-length y) (sqr z))). This is another that takes 3 and sees if the first is equal to the string length of the seconds is equal to the square of the third.

If we convert that into some non-anonymous function like (define (foo x y z) (= x (string-length y) (sqr z))), that is equivalent to writing (define foo (λ (x y z) (= x (string-length y) (sqr z)))).

Now let’s go back to our original example. How can we make that simpler with lambda?

; adder : Number -> [Number -> Number]
(define (adder n)
  (λ (x) (+ n x)))
(check-expect ((adder 3) 4) 7)

λ is even sparser than local, and as you can see, we don’t have a signature or purpose statement just for the function produced like we did with local because it’s so obvious. λ, like local, should really only be used when it’s simple enough.

Let’s design a program where it takes in a background image big-bang and draws increasingly large numbers on it.

; draw-numbers-on : Image -> Number
; Draw numbers on i
(define (draw-numbers-on-i bg)
  (big-bang 0
    [on-tick add1 1]
    [to-draw (λ (n) (overlay (text (nubmer->string i) 13 "green") bg))]))

Note what’s going on here: because bg is in scope here, we can use it in a function given to a big-bang handler via lambda. We also could have done this with local.
