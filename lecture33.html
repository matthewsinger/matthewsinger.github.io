Lecture 33: Binary Trees

Motivation: We want to be able to draw tree-shaped data

Topics to cover: binary trees, their templates, and functions that process them

Recap

Any insights from the homework? What didn’t people understand?

Tree

Suppose you’re driving, and every road is a certain amount of miles long. At the end of every road is either a fork in the road or a dead end. How would you represent this data?
; A Road is one of:
; - Number
; - (make-road Number Road Road)
; and represents a the # of miles a road is
; and the two roads at the end at the fork (in the latter case)

How does this differ from the data definitions we’ve seen so far? It recurs in two places. Let’s define examples and a template.

; A Road is one of:
; - Number
(define-struct forked [miles left right])
; - (make-forked Number Road Road)
; and represents a the # of miles a road is
; and the two roads at the end at the fork (in the latter case)
 
(define r0 5)
(define r1 (make-road 6 r0 r0))
(define r2 (make-road 10 r1 r0))
 
; road-temp : Road -> ?
(define (road-temp r)
  (cond [(number? r) ...]
        [(forked? r) (... (forked-miles r) (road-temp (forked-left r)) (road-temp (forked-right r)))]))

Draw the examples on the board to show their shape.

If you squint, you can see these roads look like trees. Data that recurs in multiple places are called trees. This kind of tree is a binary tree because it recurs in two places.
