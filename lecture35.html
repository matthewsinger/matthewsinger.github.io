Multiple complex inputs

Let’s say we have a tree with this funky shape and a list of directions:

(define-struct branch [in])
(define-struct fork [left right])
 
; A FT (FunkyTree) is one of:
; - 'end
; - (make-branch FT)
; - (make-fork FT FT)
 
; A Direction is one of:
; - 'forward
; - 'left
; - 'right

And we want to know if a list of directions is valid for the given tree. The function is processing two complex inputs: a funky tree and a list. Which template should we follow?

Well, both! There are six possiblities: 3 from FT and 2 from list, and since any combo is possible, we multiply them together.

; valid-dirs? : [List-of Direction] FunkyTree -> Boolean
; Is this list of directions valid for this funky tree?
(define (valid-dirs? lod ft)
  (cond [(and (empty? lod) (symbol? ft)) #t]
        [(and (empty? lod) (branch? ft)) #t]
        [(and (empty? lod) (fork ft)) #t]
        [(and (cons? lod) (symbol? ft)) #f]
        [(and (cons? lod) (branch? ft)) (and (symbol=? 'forward (first lod))
                                             (valid-dirs? (rest lod) (branch-in ft)))]
        [(and (cons? lod) (fork ft)) (or (and (symbol=? 'left (first od))
                                              (vaid-dirs? (rest lod) (fork-left ft)))
                                         (and (symbol=? 'right (first od))
                                              (vaid-dirs? (rest lod) (fork-right ft))))]))
If the list is empty, it’s always valid, so we can simplify the first three lines.

Let’s design remove for a BST.

; A BST (Binary Search Tree) is one of:
; - 'leaf
(define-struct node [value left right])
; - (make-node Number BST BST)
; and represents a binary tree where all of the numbers
; to the left of value are smaller than it
; and all the numbers to the right are larger than it
 
 
 
; remove/bst : Number BST -> BSt
; Remove n from bst
(define (remove/bst n bst)
  (cond [(symbol? bst) bst]
        [(node? bst)
         (cond [(= n (node-value bst)) (create-bst (node-left bst) (node-right bst))]
               [(< n (node-value bst)) (make-node (node-value bst) (remove/bst n (node-left bst)) (node-right bst))]
               [(> n (node-value bst)) (make-node (node-value bst) (node-left bst) (remove/bst n (node-right bst)))])]))
 
; create-bst : BST BST -> BST
; Create one BST from two, where all in smaller is smaller than all in larger
(define (create-bst smaller larger)
  (cond [(and (symbol? smaller) (symbol? larger)) smaller]
        [(and (symbol? smaller) (node? larger)) larger]
        [(and (node? smaller) (symbol? larger)) smaller]
        [(and (node? smaller) (node? larger))
         (make-node (node-value smaller)
                    (node-left smaller)
                    (create-best (node-right smaller) larger))]))

The disadvantage to this strategy is that it creates highly imbalanced trees. Self-balancing trees are an interesting topic, but outside the scope of this lecture.
