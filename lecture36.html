Quote

So let’s talk about a piece of syntax, ’, which will make our lives easier.

'symbol
'"String"
'5
'#t
'()
'(cat dog "bark")
'((list1) (list2))

Quote works by being applied to every expression inside of it. What would '(make-posn 3 4) or '(+ 3 4) produce?

If you want to evaluate some of the expression, use quasiquote/unquote, as in

`((make-posn 3 4) (+ 3 4) ,(make-posn 3 4) ,(+ 3 4))

So what values are possible in a quoted expression?

SExpr

Values possible from just a quote expression are called S-Expressions, abbreviated as SExpr, which stand for symbolic expressions.

; An SExpr is one of:
; - Symbol
; - Number
; - Boolean
; - String
; - [List-of SExpr]
 
; A [List-of SExpr] is one of:
; - '()
; - (cons SExpr [List-of SEXpr])

Notice what’s interesting about this data: it’s mutually recursive. Let’s write the template.

; sexpr-temp : SExpr -> ?
(define (sexpr-temp s)
  (cond [(symbol? s) ...]
        [(number? s) ...]
        [(boolean? s) ...]
        [(string? s) ...]
        [(list? s) (los-temp s)]))
 
; los-temp : [List-of Sexpr] -> ?
(define (los-temp los)
  (cond [(empty? los) ...]
        [(cons? los) (... (sexpr-temp (first los))
                          (los-temp (rest los)))]))

Both templates are calling to each other. This the first time we’ve seen something like this.
