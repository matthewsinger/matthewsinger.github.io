Insert sort

Let’s use structural recursion to sort a list of numbers.
; sort/lon : [List-of Number] -> [List-of Number]
(define (sort/lon lon)
  (cond [(empty? lon) '()]
        [(cons? lon) (insert (first lon) (sort/lon (rest lon)))]))
 
; insert : Number [List-of Number] -> [List-of Number]
(define (insert n lon)
  (cond [(empty? lon) (list n)]
        [(cons? lon) (if (<= n (first lon))
                         (cons n lon)
                         (cons (first lon) (insert n (rest lon))))]))

But this is slow. It has to go through the entire recursively built list every time. Let’s try a way to make this faster.

Quick sort

We’re going to pick a pivot element, sort those less than it, sort those bigger than it, and combine them.

; qsort : [List-of Number] -> [List-of Number]
(define (qsort lon)
  (cond [(empty? lon) '()]
        [(cons? lon) (local [(define pivot (first lon))
                             (define smaller (λ (n) (<= n pivot)))
                             (define bigger (λ (n) (> n pivot)))]
                       (append (qsort smaller) (qsort bigger)))]))

Uhoh, this doesn’t terminate! Why not? The size of the problem isn’t shrinking. When using non-structureal recursion, you have to take care to ensure your function actually terminates. The template ensures the size of the problem shrinks, but when it comes to some clever problems, you don’t always get that guarantee.

; qsort : [List-of Number] -> [List-of Number]
; Termination: Size of list shrinks in each recursive call
(define (qsort lon)
  (cond [(empty? lon) '()]
        [(cons? lon) (local [(define pivot (first lon))
                             (define smaller (λ (n) (< n pivot)))
                             (define equal (λ (n) (= n pivot)))
                             (define bigger (λ (n) (> n pivot)))]
                       (append (qsort smaller) equal (qsort bigger)))]))
