Lecture 5: Problem Solving

Motivation: We want to be able to solve problems by translating text into code and breaking down problems into subproblems

Topics to cover: helper functions

Recap

Any insights from the homework? What didn’t people understand?

Setup

Imagine the following scenario:

; A movie theater wants to determine what price to sell their tickets at
; The basic trade off is this: the lower tickets are, the less money they take in per ticket,
; but the higher the ticket, the fewer tickets are sold
 
; The owner of the theater wants to write a program that, given the cost of a ticket,
; will determine how much money is made per showing.
 
; Some data was collected, and the owner knows this:
; At 5.00 a ticket, 120 people show up on average
; For every 10 cents above 5 dollars, 15 fewer people come on average.
; The opposite is also true: for every 10 cents below 5 dollars, 15 more people come on average.
 
; Every movie showing costs the theater 180 dollars to put on, and
; for every person in attendance, it costs them 4 cents (more popcorn to clean up)
 
; So now, where do we go from here?

Think First, Code Later

First, we’re going to the board to figure out how to break this problem down.

What is the input? (ticket cost)

What is the desired output? (revenue)

What is the formula for revenue? (profit of show - cost of show)

What is the formula for profit of show? (ticket cost * #attendees)

What is the formula for cost of show? (base cost + cost-per-person * # attendees)

What is the #attendees? (base attendance - 15*(ticket cost - 5)/.1) = (base attendance - 150*(ticket cost - 5))

Put a box around which of these are constants. What is left over? Just ticket cost, which is our input. This is good because we can solve the problem with just the input.

What shows up multiple times?

Converting to code

Work through translating what’s written on the board into the following program, or something close to it:

; profit : Number -> Number
; calculate the profit for the movie theater owner at a given ticket price
(define (profit ticket-price)
  (- (revenue ticket-price)
     (cost ticket-price)))
(check-expect (profit 4) 889.2)
(check-expect (profit 5) 415.2)
(check-expect (profit 6) -180)
 
; revenue : Number -> Number
; how much money I bring in when tickets are a certain price
(define (revenue ticket-price)
  (* ticket-price
     (attendance ticket-price)))
(check-expect (revenue 4) 1080)
(check-expect (revenue 5) 600)
(check-expect (revenue 6) 0)
 
; cost : Number -> Number
; how much I spend when tickets are a certain price
(define (cost ticket-price)
  (+ BASE-COST
     (* EXTRA-COST-PER-PERSON
        (attendance ticket-price))))
(check-expect (cost 4) 190.8)
(check-expect (cost 5) 184.8)
(check-expect (cost 6) 180)
 
; attendance : Number -> Number
; the number of people who will come to the theater at a certain ticket price
(define (attendance ticket-price)
  (max 0
       (- BASE-ATTENDANCE
          (* ATTENDANCE-LOST-PER-DOLLAR
             (dollars-past-five ticket-price)))))
(check-expect (attendance 4) 270)
(check-expect (attendance 5) 120)
(check-expect (attendance 6) 0)
 
; dollars-past-five : Number -> Number
; the number of dollars by which the ticket price exceeds $5 (potentially negative)
(define (dollars-past-five ticket-price)
  (- ticket-price 5))
(check-expect (dollars-past-five 4) -1)
(check-expect (dollars-past-five 5) 0)
(check-expect (dollars-past-five 6) 1)

Lesson

Without breaking this problem into subproblems and using constants this code would be almost unreadable.

Repeat after me: Think first, code later!

Repeat after me: One task per function.
