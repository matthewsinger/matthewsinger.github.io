big-bang

Programming Language BSL

Purpose Begin to create big-bang programs.

Setup Recall the syntax for a big-bang expression:

(big-bang world0
  [to-draw drawing-function] ; mandatory
  [handler1 handler1-function] ; this and all others handlers are optional
  [handler2 handler2-function]
  ...
  [handlern handlern-function])

The general signatures and purpose statements for the functions given to big-bang handlers we’ve learned so far (to-draw, on-tick, and stop-when) are:

; to-draw-function : WorldState -> Image
; Draw the world state
 
; on-tick-function : WorldState -> WorldState
; Advance the world state by one tick
 
; stop-when-function : WorldState -> Boolean
; Should the program stop?

Remember not to use WorldState in your signatures and instead the data definition (whether user defined or primitive) specific to your program, and to specify the purpose statements to your program.

Remember to wrap your calls to big-bang in some main function.

Remember that some handlers can take some arguments besides just the function:

  • on-tick can take a number as a second argument which tells big-bang how many seconds should occur before the next tick.

  • stop-when can take a second argument which tells big-bang how to draw the last frame. Usually, this is just going to be the same as the function you give to to-draw. If you don’t give stop-when this second function, your visuals will usually stop one frame short.

Exercises

Exercise 1 Your friend Hephaestus tells you he’s written an incredible animation with the following function:
; hephaestus : Nat -> Image
; Draw the nth frame of an amazing animation

Hephaestus, however, wants to know how to run his amazing animation by using big-bang. In a block comment, write a big-bang expression that is functionally equivalent to (animate hephaestus).

Exercise 2 Design a function double which given some string s, will display s, then ss, then ssss, then ssssssss, etc., in the center of the screen. It should stop after the text has gone past the edge of the screen.

Exercise 3 Design a data definition which describes a finite collection of shapes, then design a function which will cycle through images of these shapes.

Exercise 4 Design a function countdown that will visually count down from some given natural number until it hits 0, at which point the program should stop and the screen should display "Done!".

Exercise 5 The Collatz conjecture is an open problem in mathematics. Consider the following procedure:

  1. Let n be some positive integer.

  2. If n is 1, stop.

  3. If n is even, divide it by 2. This is the new n, and go back to step 2.

  4. If n is odd, multiply it by 3 and add 1. This is the new n, and go back to step 2.

For example, if n is initially 12, the following sequence describes the values n would be: [12, 6, 3, 10, 5, 16, 8, 4, 2, 1].

The Collatz conjecture states that for any initial n, this process will always stop (that is, will eventually reach step 2 where n is 1). While this has been shown to be true for all n up to (* 87 (expt 2 60)), which is roughly 87 quintillion (a big number!), we do not know if it’s true for all positive integers.

Design the function collatz, which given a positive integer, will display each number generated in the process described above. So if collatz is given 12, it should show the sequence above, one number at a time, and then stop once it reaches 1.
