Nested Data

Programming Language BSL

Purpose Design functions with nested data.

Setup Remember that if a data definition refers to a complex (non-primitive) data type, the template for that data definition must refer to the data definition for the referenced, complex data type. When designing functions, a template calling another template is a sign to call a helper function that processes that other type of data.

Exercises

Exercise 1 Consider the following data definition:
; A CSG (CoinShuffleGame) is a (make-csg CoinOrFalse CoinOrFalse CoinOrFalse)
(define-struct csg [left middle right])
; and represents the three cups in a coin shuffle game, and what is under them
 
; A CoinOrFalse is one of:
; - #false
; - Number
; and represents either no coin or the coin's monetary value
 
; A Guess is one of:
; - "left"
; - "middle"
; - "right

  1. Provide examples and templates for the above data definitions.

  2. Design a function, shuffle-right, which moves all cup values in a CSG to the right. The right cup’s contents should loop back to the left cup.

  3. Design a function, which given a CSG and a Guess, outputs the monetary value of the guessed cup. No coinage is worth 0.

  4. Design the function inflation, which given a CSG and a number, adds that monetary value to all of the coins in the cup, and leaves empty cups as is.

Exercise 2 Using big-bang, design a program which will display two circles. Each circle should grow up to a certain size, then shrink back to 0, then repeat. While one circle is growing, the other should be shrinking. Use the data definition below to help, but you will need to design your own to run the whole program.
; A Circle is a (make-circ Nat Boolean)
(define-struct circ [radius growing?])
; and represents a circle's radius
; and whether or not it is currently growing or shrinking
