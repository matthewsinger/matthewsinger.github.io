Complex Lists

Programming Language BSL

Purpose Design a program using lists.

Setup N/A

Exercises

Exercise 1 A slideshow consists of a list of slides. A slide has a title, a list of bullet points that are currently being shown, and a list of bullet points that have yet to be shown. This data definition captures such a scenario:
; A Slideshow is one of:
; - empty
; - (cons Slide Slideshow)
; and represents an ordered slideshow
 
; A Slide is a (make-slide String LoS LoS)
(define-struct slide [title shown hidden])
; and represents a slide's title,
; what bullets have been shown (top to bottom)
; and which are hidden (top to bottom)
 
; An LoS (List of Strings) is one of:
; - empty
; - (cons String LoS)

Design a program using big-bang which takes in a Slideshow and runs the slideshow. Slides should always display their title and whatever bullets are not hidden. The user should be able to advance the slideshow whether by clicking or by pressing the right arrow key. Advancing the slideshow when there are no more hidden bullets on the current slide should move the slideshow to the next slide. Advancing the slideshow when there are more hidden bullets should display the first hidden bullet at the bottom of what was already shown. The program should display a black screen when the slideshow ends, which should also end the program.

Hint: break the above paragraph down by assigning different sentences to the relevant big-bang handler and/or wishlist a helper function. Work on one handler at a time. If different handlers serve the same purpose, you should use a helper function to reduce repeated code as much as possible.

Since big-bang programs are of a fixed width and height, it is fine if long slides go off screen, but you should design your program so you can fully see the examples you’re testing on.
