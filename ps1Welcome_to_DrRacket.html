Welcome to DrRacket

Programming Language BSL

Purpose Learn to anticipate and parse errors and evaluation, understand terminology, convert arithmatic to expressions

Setup If you haven’t already, download and install DrRacket.

Once you open DrRacket, set the language to BSL. Go to Language > Select Language > Beginning Student > OK. Double check that DrRacket says Beginning Student Language in the lower left hand corner.

Exercises

Exercise 1 In programming, errors are inevitable. Anticipating errors and understanding error messages will be an essential part of your growth as a developer.

Therefore, for each of the following snippets of code, leave a block comment with:

  1. Just that piece of code at the top

  2. The error you expect that will occur when you run just that piece of code in the definitions window

  3. The actual error that occurs when you run just that piece of code in the definitions window (you can open a separate tab with File > New Tab to run these broken snippets)

  4. If the error you expected is significantly different from the actual error, explain why you think this is

  5. If you don’t understand the error that is produced, say which words or clauses you don’t understand

Remember, a block comment can span multiple lines and starts with #| and ends with |#.

  • (

  • )

  • ()

  • )(

  • +

  • (/ 5 0)

  • (substring "cat" 0 5)

  • (string-length "apple" "bee")

  • (3 4)

  • (+ 2 "dog")

Exercise 2 For each of the following expressions, leave a block comment indicating how you think the expression will be evaluated. For example:

(+ 3 (+ 4 5)) will next evaluate to (+ 3 9) and then 12. Each new line of the block comment should indicate a new step in the evaluation. Then, run the expression in the stepper in a new window, and below your prediction, show how it actually evaluates. If your prediction differs from the actual result, leave a comment explaining how.

  • (string-append (substring "chandelier" 3 (- 9 5)) "orc")

  • (+ 1 2 3)

  • (+ (+ 1 (+ 2 3) 4) (+ 5 6))

Exercise 3 For each of the following arithmetic expressions, leave a one-line comment with the expression, and below it, convert it into a proper BSL expression.

Remember, a one-line comment starts with a semicolon.

  • 3 - (1 + 2)

  • (3 * 7) - (1 + 2)

  • 3 - (1 + (5 * 6))

Exercise 4 Consider the expression:

(+ (string-length "cat") 12 (* 4 5))

Answer these questions in a block comment:

  • What is the name of the outermost function?

  • What are the names of all the functions being called?

  • How many arguments are being passed to +?
