Abstracting Code

Programming Language ISL

Purpose Abstract similar code.

Setup Abstracting two or more similar functions requires four steps:

  1. Identify the differences between the functions.

  2. Turn those differences into parameters in an abstracted function.

  3. Redefine the original functions in terms of the new function.

  4. Ensure old tests still pass.

Remember that function types are written in brackets with an arrow. For example, a function that took a number and a string and produced a boolean would have the type:

; [Number String -> Boolean]

This will be useful when writing signatures.

Be sure to change your language to Intermediate Student Language!

Exercises

Exercise 1 For this exercise, you may assume the standard definition for ListofPosn and Posn, and do not need to define them or their templates. You can if you want to, though.

Consider the following functions:
; sum-x-coords : ListOfPosn -> Number
; Sum all the x-coordinates in the list of positions
(define (sum-x-coords lop)
  (cond [(empty? lop) 0]
        [(cons? lop) (+ (posn-x (first lop))
                        (sum-x-coords (rest lop)))]))
(check-expect (sum-x-coords empty) 0)
(check-expect (sum-x-coords (list (make-posn 3 4) (make-posn 5 12))) 8)
 
; mult-distances : ListOfPosn -> Number
; Multiply all the distances from each position to the origin
(define (mult-distances lop)
  (cond [(empty? lop) 1]
        [(cons? lop) (* (distance-to-origin (first lop))
                        (mult-distances (rest lop)))]))
(check-expect (mult-distances empty) 1)
(check-expect (mult-distances (list (make-posn 3 4) (make-posn 5 12))) 65)
 
; distance-to-origin : Posn -> Number
; Produces the distance from this position to the origin
(define (distance-to-origin p)
  (sqrt (+ (sqr (posn-x p))
           (sqr (posn-y p)))))
(check-expect (distance-to-origin (make-posn 3 4)) 5)

  1. Abstract sum-x-coords and mult-distances.

  2. Use your new function to define biggest-difference, which produces the largest difference between a posns’s x and y values in a list of posns. Note that the difference should always be taken as an absolute value, whether the x or y value is bigger.

Exercise 2 Design a function which takes a non-empty list of strings and a function which takes two strings and outputs a boolean that indicates whether or not the first string comes "before" the second one. This function should output the string that comes earliest in the non-empty list according to the given function.

Use this to define three other functions:

  • One that produces the string that comes earliest lexographically in a non-empty list

  • One that produces the string that comes last lexographically in a non-empty list

  • One that produces the last string in the non-empty list

Here, earliest lexographically effectively means what comes first in the dictionary.

Hint: the pre-defined string<? function will be helpful.
