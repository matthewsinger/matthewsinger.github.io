Paramtetic Data

Programming Language ISL

Purpose Use parametric data to create even more abstract code.

Setup In a signature, we indicate which types are abstract with parenthesis, e.g.:

; to-string : (X) [X -> String] X -> String
; Convert x to a string with f
(define (to-string x f)
  (f x))

Remember to use [List-of X] notation in your signatures.

Exercises

Exercise 1 Sometimes, we want to bin items by a certain property. For example, let’s say we had the following definition for cups:
; A Cup is a (make-cup NNN String String)
(define-struct cup [oz color material])
; and represents a cup's capacity in fluid ounces, color, and material
 
(define cup1 (make-cup 10 "brown" "wood"))
(define cup2 (make-cup 8 "brown" "ceramic"))
(define cup3 (make-cup 10 "red" "plastic"))
(define cup4 (make-cup 6 "clear" "plastic"))

We could bin by color, in which case we’d have something like:

  • brown - cup1, cup2

  • red - cup3

  • clear - cup4

Alternatively, we could bin by material, and we’d have something like:

  • wood - cup1

  • ceramic - cup2

  • plastic - cup3, cup4

Note that this is the same binning as:

  • wood - cup1

  • plastic - cup4, cup3

  • ceramic - cup2

The order of bins produced and the order in the individual bin does not matter; only which bins are created and the correct elements going to the correct bins.

  1. Design a function bin which takes in 4 arguments - a list of elements, a key extractor, a key equivalence relation, and an element equivalence relation - and produces a binning.

    A key extractor is a function that extracts information from an element to determine what bin it should belong to. In the first example given, it would be a function that takes a cup and produces its color.

    An equivalence relation is a function which takes two elements and produces a boolean indicating whether or not they are equal. In the first example, the key equivalence relation would be a function which tells us if two colors are equal, and the element equivalence relation would be a function which tells us if two cups are equal.

    A binning is a structure that allows us to capture a list of keys and the elements that belong to that key. Exactly what data definition you use is up to you to determine.

    Hint: you are going to want to use helper functions, and you may want to complete the step below before attempting this one. Also, think carefully about the signature of bin and use it to help you. It’s complex, but isn’t significantly different from the signatures we’ve seen in class.

  2. As two bins can be equivalent but may not have the same structure, check-expect is not very helpful here. Design a function which takes two binnings, a key equivalence relation and an element equivalence relation, and determines if the two binnings are equal or not. Use this function to test the output of bin by testing to see if the output of bin is an equivalent binning to one you would expect it to be. Be sure to thoroughly test this function, too.

    Note: you can assume all of the elements in the original input list that were binned are unique according to the element equivalence relation.
