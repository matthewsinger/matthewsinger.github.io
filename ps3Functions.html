Functions

Programming Language BSL

Purpose Convert algebra to code, practice using the design recipe for functions, derive signatures

Setup Recall the process for writing functions with the design recipe: signature, purpose statement, stub, tests, code. Note that your code should replace the body of your stub and there is never a need to leave a stub lying around in a submission.

From here on in, all of your functions must come with a signature, purpose statements, and tests, unless stated otherwise. We specifically design functions as opposed to writing them as a reminder that the design of our programs are paramount.

Don’t forget to add (require 2htdp/image) at the top of your file and to be sure you’re using BSL.

Exercises

Exercise 1 Consider the function times-itself and paste it into your code:

; times-itself : Number -> Number
; x squared
(define (times-itself x)
  (+ x x))
(check-expect (times-itself 0) 0)
(check-expect (times-itself 2) 4)

All of the tests pass, but upon careful inspection, the code is wrong! Leave a comment explaining why times-itself is wrong, provide a test that demonstrates why it’s wrong, and then go back and fix it so your test that showed it was broken now passes.

This kind of test is called confirmation testing, which confirms a bug in the code that was discovered has been succesfully squashed.

Exercise 2 Convert the following algebraic functions into proper BSL functions:

  • f(x) = (5x + 2)*(3x + 4)

  • g(a, b) = 3a + 2b

Exercise 3 Design the function image-area, which takes an image and outputs its area in pixels.

Hint: Recall the definition for the area of a rectangle, and look at the image docs to see if there are any functions that might help.

Exercise 4 Define a constant by the name of GREETING and use it in a function called greeting, which given someone’s name and title, outputs a greeting. Fun greetings like "Merry day to you, ____" are encouraged!

Exercise 5 Design a function which replicates a string n times, where n is the length of the string. Be sure to include a test on the empty string, "".

Exercise 6 Copy the following function into your document and give it a signature. Confirm this signature by writing a check-expect which passes. As this code is nonsense, do not provide a purpose statement.
(define (nonsense a b c d e)
  (+ (string-length (substring c (- b d) a)) e))

Exercise 7 Design a function latter which takes two strings and outputs the second one given to the function. For example, this test should pass: (check-expect (latter "apple" "bee") "bee")

Beneath this and your other tests, write a bad test for this function and leave a comment explaining why it’s bad. A bad test is one that does not actually test the functionality of the code.
