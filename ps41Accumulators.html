Accumulators

Programming Language ISL+

Purpose To begin designing code with accumulators.

Setup An accumulator is a piece of data that is kept track of between recursive calls that would otherwise be forgotten. The four questions to ask yourself when using an accumulator are:

  1. What does the accumulator represent?

  2. What should its initial value be?

  3. How should it be updated between recursive calls?

  4. How should it be used in the final (non-recursive) answer?

For the following exercises, you should use an accumulator.

Exercises

Exercise 1 Design a function which appends every string in a list of strings together. This function should be tail recursive (i.e. the recursive call is never passed to another function).

Exercise 2 Design a function which takes a list of numbers that represent relative distances and converts them to absolute distances. For example, '(1 3 5) as relative distances translates to '(1 4 9) in absolute distances.

Exercise 3 Design a function which takes a list of symbols and eliminates consecutive duplicate elements. For example, eliminating the consecutive duplicate elements of '(a a b c c c a b b) should produce '(a b c a b).
