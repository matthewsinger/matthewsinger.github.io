Accumulators Part Two

Programming Language ISL+

Purpose To design more interesting problems with accumulators.

Setup An accumulator is a piece of data that is kept track of between recursive calls that would otherwise be forgotten. The four questions to ask yourself when using an accumulator are:

  1. What does the accumulator represent?

  2. What should its initial value be?

  3. How should it be updated between recursive calls?

  4. How should it be used in the final (non-recursive) answer?

For the following exercises, you should use at least one accumulator.

Exercises

Exercise 1 Design a data definition to represent simple lambda expressions (a variable, a nameless function of one argument, or a function application) and then design a function which returns the list of all of the unbounded variables that occur in the lambda. Your result should not contain duplicates.

Exercise 2 Design a function biggest-consecutive-streak which determines which symbol in a non-empty list of symbols has the biggest consecutive streak. For example, 'c has the longest consecutive streak in '(b a a b c c c b b), despite not appearing the biggest number of times. Break ties by choosing the symbol that appeared earlier in the list.

Exercise 3 Design a function mode that takes a non-empty list of strings and returns the strings that appear most in the list. Use the techniques we have learned thusfar to calculate this as efficiently as possible.

Hint: the modes of a list of strings is very closely related to the word count problem.
