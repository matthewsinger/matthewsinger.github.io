Conditional Programming

Programming Language BSL

Purpose Use booleans to write conditional programs with if and cond

Setup Remember, the general syntax for if and cond are:

(if test then else) ; if test is true, evaluates then, otherwise evaluates else
(cond [test1 answer1]
      [test2 answer2]
      [test3 answer3]
      ...
      [testx answerx]
      [else answer-else]) ; the else clause is optional!
; cond evaluates and returns the FIRST answer for the test that evaluates to true
; doesn't evaluate any tests past the test that passes

You should use cond when there are 3 or more tests. It is fine to use cond when there are only 2, but if is often shorter.

Exercises

Exercise 1 Design a function orientation that takes an image and says if it’s "landscape" (wider than it is tall), "portrait" (taller than it is wide), or "square".

Exercise 2 Design a function absolute-value which outputs the absolute value of a number (its distance from zero). Do not use abs.

Exercise 3 Design a function perfect-square? which determines if the square root of a non-negative number (we abbreviate that type as NNN) is an integer.

Predicates like string?, even?, integer?, or perfect-square? are often named with a question mark at the end. It is good style to have the purpose statement of a predicate be in the form of a question. A predicate is a function that takes one input and outputs a boolean.

Exercise 4 Consider the following code:
; 1-or-0 : Boolean -> Number
; Output 1 if b is true, else 0
(define (1-or-0 b)
  (if (if b #true #false) 1 0))
(check-expect (1-or-0 #t) 1)
(check-expect (1-or-0 #f) 0)
Explain in a comment why (if b #true #false) is a silly expression, and adjust the function so that it is less silly.

Exercise 5 Design a function both-true? that takes two booleans and returns #true only if both booleans are #true. The body of this function should only use if, the names of the parameters, #true, and #false (though you may not need all of these).

Exercise 6 Design a function either-true? that takes two booleans and returns #true if either of the booleans are #true. The body of this function should only use if, the names of the parameters, #true, and #false (though you may not need all of these).

Exercise 7 Design a function negate that takes one boolean and returns the opposite boolean. The body of this function should only use if, the name of the parameter, #true, and #false (though you may not need all of these).

Exercise 8 Design a function divisible-by? that takes two numbers and determines if the first is evenly divisible by the second.

Exercise 9 Design the function leap-year? which determines if a natural number represents a leap year in the Gregorian calendar.

Exercise 10 Design an animation grow-and-shrink which should behave as growing did from homework 4, but instead of going from the edge of the screen to tiny instantly, should shrink back to tiny and then loop again. quotient will likely be helpful.
